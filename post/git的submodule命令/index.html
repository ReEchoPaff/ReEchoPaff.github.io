Git的submodule命令

git submodule

git submodule add [url] [path]
url：远程仓库地址

path：本地子模块存放地址

1.添加子模块

首先创建父项目parentRepo，并添加sub1Repo子模块。

1
2
3
4
5

mkdir parentRepo
cd parentRepo/
git init

git submodule add ~/Code/blog/exp1Repo sub1Repo

git status 看到如下信息

1
2
3
4
5
6
7
8
9

位于分支 master

尚无提交

要提交的变更：
  （使用 "git rm --cached <文件>..." 以取消暂存）

	新文件：   .gitmodules
	新文件：   sub1Repo

cat .gitmodules 看到submodule的路径以及对应的远程仓库地址（此处远程仓库也可以github仓库）

1
2
3

[submodule "sub1Repo"]
	path = sub1Repo
	url = /home/reguo/Code/blog/exp1Repo

提交这两个文件

1
2

git add .
git commit -m 'add sub1Repo'

查看子模块的commit id

1
2
3

~/Code/blog/parentRepo/.git/modules/sub1Repo$ cat packed-refs 
# pack-refs with: peeled fully-peeled sorted 
adc1415eb37f2047b9514605420ce1554c91528e refs/remotes/origin/master

2.克隆带有submodule的项目

两种方式

1. 先克隆后更新

1
2
3
4

git clone /home/reguo/Code/blog/parentRepo/.git
cd parentRepo/
git submodule init
git submodule update

2. 采用递归参数 –recurse-submodules

1
2
3
4
5
6
7

git clone /home/reguo/Code/blog/parentRepo/.git --recurse-submodules 
正克隆到 'parentRepo'...
完成。
子模组 'sub1Repo'（/home/reguo/Code/blog/exp1Repo）未对路径 'sub1Repo' 注册
正克隆到 '/home/reguo/Code/blog/exp2Repo/parentRepo/sub1Repo'...
完成。
子模组路径 'sub1Repo'：检出 'adc1415eb37f2047b9514605420ce1554c91528e'

可以连同子仓库一起克隆下来。

3.修改更新submodule

直接在主项目的submodule下修改

切换到子模块内修改代码，然后在子模块内提交，此时的commit id是本地最新的。将此提交push的远程子仓库，会报错，显示无法push。需要在远程的子仓库内修改配置，添加如下：

1
2

[receive]
	denyCurrentBranch = ignore

此时再次push，显示成功，这种情况下并不会更新到远程的子仓库（commit id会更新）。但是如果有别的仓库是克隆自这个主仓库，在别的仓库下执行

1
2

git pull
git submodule update

会获得这次子仓库的更新。
另外当远程子仓库有更新时，执行git submodule update，也可以进行更新（但是会覆盖掉本地对子仓库的修改）。

4.删除submodule

执行如下命令

1

git submodule deinit [子模块名]

会清除子模块内容

1
2
3

git rm --cached [子模块名]
git submodule
git commit -am 'Remove a submodule.[子模块名]'

删除.gitmodules中记录的模块信息（–cached选项清除.git/modules中的缓存), 之后可以在使用 rm 命令删除对应的子模块文件夹和.gitmodule文件.

5.修改子模块的url

修改.gitmodule内对应模块的url， 使用git submodule sync更新到.git/config文件, git commit -am "Update submodule url." 提交变更.
